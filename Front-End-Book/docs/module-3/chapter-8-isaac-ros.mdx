## 1. Isaac ROS Architecture Overview

### 1.1 What is Isaac ROS?

**Isaac ROS** is NVIDIA's suite of ROS 2 packages providing GPU-accelerated perception algorithms for robotics applications. Unlike traditional CPU-only ROS packages, Isaac ROS leverages NVIDIA GPUs to achieve real-time performance on edge robots.

**Key capabilities**:
- **NVIDIA TensorRT**: Optimized inference engine for deep learning models
- **NVIDIA CUDA**: GPU compute for feature extraction and image processing
- **GXF (Graph Execution Framework)**: Dataflow-based execution model for perception pipelines
- **ROS 2 Integration**: Standard ROS 2 topics, services, and launch files
- **Jetson Deployment**: Optimized for NVIDIA Jetson edge devices (Orin, AGX)

**Performance improvements** over CPU implementations:
- Visual SLAM: **5-10x speedup**
- Depth perception: **3-5x speedup**
- Semantic segmentation: **10-15x speedup**
- Real-time capable on edge hardware (Jetson Orin: 200+ FPS)

### 1.2 Architecture Stack

```
ROS 2 Applications (Nav2, MoveIt, etc.)
    ↑ (ROS 2 topics: /odometry, /camera/depth, etc.)

Isaac ROS Perception Nodes
    ├─ Isaac ROS VSLAM
```

## 2. Visual SLAM (V-SLAM) Pipeline

### 2.1 VSLAM Fundamentals

**Visual SLAM** estimates camera pose and builds a map using only camera images. The process:

```
Input Image Sequence
    ↓
Feature Detection
    └─ Extract distinctive keypoints (SIFT, ORB, SuperPoint)
    ↓
Feature Matching
    └─ Match features between consecutive frames
    ↓
Motion Estimation
    └─ Compute camera rotation (R) and translation (t)
    ├─ Essential matrix computation
    ├─ Camera pose recovery
    ↓
Map Update
    ├─ Triangulate 3D point positions
    ├─ Update pose graph
    ↓
Loop Closure Detection
    └─ Recognize previously visited locations
    ↓
Output Odometry & Map
    ├─ /odometry/visual_odometry (tf: camera_odom → camera)
    ├─ /map (point cloud of environment)
```

### 2.2 Feature Detectors

Different algorithms for different use cases:

| Detector | Speed | Robustness | GPU Support |
|----------|-------|-----------|------------|
| **ORB** | Fast (1000+ FPS) | Good in texture | Software |
| **SIFT** | Medium (100 FPS) | Excellent (scale/rotation) | CUDA available |
| **SuperPoint** | Medium (100 FPS) | Excellent (learning-based) | TensorRT optimized |
| **SURF** | Fast (500 FPS) | Good | Limited GPU |

Isaac ROS VSLAM typically uses **SIFT or SuperPoint** for best accuracy with GPU acceleration.

## 3. Sensor Fusion

### 3.1 Fusion Strategy

Combining depth + IMU improves localization:

```
Visual Odometry (from VSLAM)
    ├─ 6 DOF pose (accurate position, drifts over time)
    ↓
Depth Measurements
    ├─ Accurate short-range distance (0.1-5m)
    ↓
IMU (Accelerometer + Gyroscope)
    ├─ Fast rotation rate (high frequency)
    ├─ Short-term attitude (drifts over time)
    ↓
Kalman Filter / Complementary Filter
    └─ Fuse: takes best from each sensor
    ↓
Fused Odometry
    └─ Accurate, stable, real-time
```

### 3.2 Extended Kalman Filter (EKF)

**EKF State**:
```
x = [px, py, pz, vx, vy, vz, qx, qy, qz, qw]  (10-dimensional)
    └─ 3D position, 3D velocity, orientation quaternion
```

**Measurement updates**:
- **Visual odometry**: Updates position with low covariance
- **Depth**: Constrains forward distance
- **IMU**: Updates velocity and orientation continuously

**Covariance matrices**:
```
Q = Process noise (how much we trust dynamics)
R_vision = Measurement noise for VSLAM (typically 0.01-0.1)
R_depth = Measurement noise for depth (typically 0.05-0.5)
R_imu = Measurement noise for IMU (typically 0.001-0.01)
```

### 3.3 Sensor Fusion Implementation

```python
#!/usr/bin/env python3
"""Sensor fusion using Extended Kalman Filter."""

import numpy as np
from dataclasses import dataclass


@dataclass
class FusionState:
    """Kalman filter state: [position, velocity, orientation]."""
    position: np.ndarray = np.zeros(3)  # x, y, z
    velocity: np.ndarray = np.zeros(3)  # vx, vy, vz
    orientation: np.ndarray = np.array([0, 0, 0, 1])  # quat xyzw


class SensorFusionEKF:
    """Extended Kalman Filter for multi-sensor fusion."""

    def __init__(self, dt: float = 0.01):
        """Initialize EKF."""
        self.dt = dt
        self.state = FusionState()

        # Covariance matrices
        self.P = np.eye(10) * 0.1  # State covariance
        self.Q = np.eye(10) * 0.001  # Process noise
        self.R_vision = np.eye(6) * 0.05  # Vision measurement noise
        self.R_imu = np.eye(3) * 0.001  # IMU measurement noise

    def predict(self, dt: float = None):
        """⏱️ Predict step using constant velocity model."""
        if dt is None:
            dt = self.dt

        # Update position: p = p + v * dt
        self.state.position += self.state.velocity * dt

        # Covariance prediction: P = F * P * F^T + Q
        F = self._jacobian_process(dt)
        self.P = F @ self.P @ F.T + self.Q

    def update_vision(self, pose_vision: np.ndarray):
        """🎥 Update with VSLAM measurement."""
        # Innovation: z - h(x)
        innovation = pose_vision - self.state.position

        # Kalman gain
        H = np.eye(3, 10)  # Observation matrix (position only)
        S = H @ self.P @ H.T + self.R_vision[:3, :3]
        K = self.P @ H.T @ np.linalg.inv(S)

        # State update
        delta_x = K @ innovation
        self.state.position += delta_x[:3]
        self.state.velocity += delta_x[3:6]

        # Covariance update: P = (I - K*H) * P
        I = np.eye(10)
        self.P = (I - K @ H) @ self.P

    def update_imu(self, accel: np.ndarray, gyro: np.ndarray):
        """📟 Update with IMU measurements."""
        # Update velocity from acceleration
        self.state.velocity += accel * self.dt

        # Update orientation from angular velocity
        # (Simplified; use quaternion integration in practice)

    def get_state(self) -> FusionState:
        """Get current fused state."""
        return self.state

    def _jacobian_process(self, dt: float) -> np.ndarray:
        """Compute Jacobian of process model."""
        F = np.eye(10)
        F[0:3, 3:6] = np.eye(3) * dt  # dp/dv
        return F


class ComplementaryFilter:
    """Simpler alternative to EKF using complementary filter."""

    def __init__(self, alpha: float = 0.95):
        """
        Initialize complementary filter.

        Args:
            alpha: Weight for low-frequency data (vision)
                  (1-alpha) weight for high-frequency data (IMU)
        """
        self.alpha = alpha
        self.position = np.zeros(3)
        self.velocity = np.zeros(3)

    def update(self, vision_pos: np.ndarray, imu_vel: np.ndarray):
        """
        Update with vision and IMU data.

        Complementary filter formula:
        output = alpha * low_freq + (1-alpha) * high_freq
        """
        # Vision (low-frequency, accurate but slow)
        # IMU (high-frequency, drifts but fast)
        self.position = self.alpha * vision_pos + (1 - self.alpha) * self.position
        self.velocity = (1 - self.alpha) * imu_vel + self.alpha * self.velocity
```

```python
class GPUMonitor:
    def __init__(self):
        self.device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

    def log_memory(self, tag=""):
        """Log GPU memory usage."""
        if torch.cuda.is_available():
            allocated = torch.cuda.memory_allocated() / 1e9  # Convert to GB
            reserved = torch.cuda.memory_reserved() / 1e9
            print(f"{tag}")
            print(f"  Allocated: {allocated:.2f} GB")
            print(f"  Reserved: {reserved:.2f} GB")

    def profile_function(self, func, *args, **kwargs):
        """Profile a function for GPU memory and time."""
        import time
        torch.cuda.reset_peak_memory_stats()
        torch.cuda.synchronize()

        start = time.time()
        result = func(*args, **kwargs)
        torch.cuda.synchronize()
        elapsed = time.time() - start

        peak_memory = torch.cuda.max_memory_allocated() / 1e9
        print(f"Function: {func.__name__}")
        print(f"  Time: {elapsed*1000:.2f} ms")
        print(f"  Peak memory: {peak_memory:.2f} GB")

        return result
```

## 5. Custom Perception Nodes

### 5.1 Developing Custom Isaac ROS Nodes

Isaac ROS provides a framework for building custom perception nodes that leverage GPU acceleration while integrating seamlessly with ROS 2.

**Custom Node Structure**:
```python
#!/usr/bin/env python3
"""Custom Isaac ROS perception node template."""

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, CameraInfo
from std_msgs.msg import Float32MultiArray
from cv_bridge import CvBridge
import torch
import numpy as np


class CustomPerceptionNode(Node):
    """
    Template for custom GPU-accelerated perception node.

    Demonstrates:
    - Image subscription and conversion
    - GPU processing with PyTorch
    - Output publication
    - Error handling and logging
    """

    def __init__(self):
        super().__init__('custom_perception_node')

        # Parameters
        self.declare_parameter('model_path', '/path/to/model.pt')
        self.declare_parameter('gpu_device', 0)
        self.declare_parameter('input_size', 640)
        self.declare_parameter('confidence_threshold', 0.5)

        self.model_path = self.get_parameter('model_path').value
        self.gpu_device = self.get_parameter('gpu_device').value
        self.input_size = self.get_parameter('input_size').value
        self.confidence_threshold = self.get_parameter('confidence_threshold').value

        # GPU setup
        self.device = torch.device(f'cuda:{self.gpu_device}' if torch.cuda.is_available() else 'cpu')
        self.get_logger().info(f"Using device: {self.device}")

        # Load model
        self.model = torch.jit.load(self.model_path).to(self.device)
        self.model.eval()

        # Bridge
        self.bridge = CvBridge()

        # Subscriptions and publications
        self.image_sub = self.create_subscription(
            Image,
            'image_in',
            self.image_callback,
            10
        )

        self.result_pub = self.create_publisher(
            Float32MultiArray,
            'perception_output',
            10
        )

        self.get_logger().info("Custom perception node initialized")

    def image_callback(self, msg: Image):
        """Process incoming image."""
        try:
            # Convert ROS Image to OpenCV
            cv_image = self.bridge.imgmsg_to_cv2(msg, 'bgr8')

            # Preprocess
            input_tensor = self.preprocess(cv_image)

            # GPU inference
            with torch.no_grad():
                output = self.model(input_tensor)

            # Postprocess
            results = self.postprocess(output)

            # Publish results
            self.publish_results(results, msg.header)

        except Exception as e:
            self.get_logger().error(f"Error in image callback: {e}")

    def preprocess(self, cv_image: np.ndarray) -> torch.Tensor:
        """Preprocess image for model."""
        # Resize
        import cv2
        resized = cv2.resize(cv_image, (self.input_size, self.input_size))

        # Normalize (ImageNet standards)
        normalized = resized.astype(np.float32) / 255.0
        normalized = (normalized - 0.485) / 0.229  # Simplified normalization

        # Convert to tensor and move to GPU
        tensor = torch.from_numpy(normalized).unsqueeze(0).permute(0, 3, 1, 2)
        tensor = tensor.to(self.device)

        return tensor

    def postprocess(self, output: torch.Tensor) -> dict:
        """Postprocess model output."""
        results = {
            'detections': [],
            'confidence': [],
            'class_ids': []
        }

        # Example: object detection output
        predictions = output[0]  # Batch size 1

        for i, pred in enumerate(predictions):
            conf = pred[4].item()
            if conf > self.confidence_threshold:
                results['confidence'].append(conf)
                results['detections'].append(pred[:4].cpu().numpy())
                results['class_ids'].append(int(pred[5].item()))

        return results

    def publish_results(self, results: dict, header):
        """Publish results as ROS message."""
        msg = Float32MultiArray()
        msg.data = np.concatenate([
            np.array(results['confidence']),
            np.array(results['class_ids'], dtype=np.float32)
        ]).tolist()

        self.result_pub.publish(msg)


def main(args=None):
    """Main entry point."""
    rclpy.init(args=args)
    node = CustomPerceptionNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 5.2 ROS 2 Interface Design

**Best practices for perception node interfaces**:

1. **Input topics**:
   - Primary: `/camera/image_rect` (rectified camera image)
   - Optional: `/camera/camera_info` (camera intrinsics)
   - Optional: `/imu/data` (for temporal alignment)

2. **Output topics**:
   - `/perception/detections` (detection results)
   - `/perception/confidence` (confidence scores)
   - `/perception/timing` (execution timing info)
   - `/perception/debug` (debug visualizations)

3. **Services**:
   - `/perception/reset` - Reset internal state
   - `/perception/set_threshold` - Adjust confidence threshold
   - `/perception/get_performance` - Query performance stats

4. **Parameters**:
   - Model path, GPU device, batch size
   - Input dimensions, normalization constants
   - Threshold values, timeouts

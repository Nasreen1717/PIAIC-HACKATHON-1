# Phase 1 Data Model: Module 4 - Vision-Language-Action (VLA)

**Date**: 2026-01-26 | **Feature**: 004-vla-capstone | **Branch**: 004-vla-capstone

## Overview

This document defines the core data structures exchanged between Whisper, LLM, and ROS 2 components. All entities are serializable to ROS 2 messages or Python dataclasses.

---

## Entity Definitions

### 1. VoiceCommand

**Purpose**: Represents a transcribed audio input from Whisper.

**Attributes**:
- `id` (string): Unique identifier (UUID)
- `timestamp` (datetime): When audio was captured
- `raw_audio_path` (string): Path to WAV file (for debugging/replay)
- `transcript` (string): Transcribed text from Whisper
- `confidence_score` (float): Whisper confidence (0.0-1.0)
- `duration_seconds` (float): Audio duration
- `language` (string): Detected language (default: "en")
- `model_version` (string): Whisper model used (e.g., "base", "small")

**Validation Rules**:
- `transcript` must be non-empty if `confidence_score` >= 0.7
- `confidence_score` < 0.5 triggers rejection (not forwarded to LLM)
- `duration_seconds` must be > 0.5 and < 300 (5 minutes)

**State Transitions**:
- `CAPTURED` → `PROCESSING` → `SUCCESS` (high confidence) or `REJECTED` (low confidence)

**Example**:
```json
{
  "id": "voice-cmd-001",
  "timestamp": "2026-01-26T10:30:00Z",
  "transcript": "bring me the red cube from the table",
  "confidence_score": 0.95,
  "duration_seconds": 3.2,
  "language": "en",
  "model_version": "base"
}
```

**ROS 2 Message Equivalent**:
```yaml
# custom_msgs/VoiceCommand.msg
string id
builtin_interfaces/Time timestamp
string transcript
float64 confidence_score
float64 duration_seconds
string language
string model_version
```

### 2. TaskPlan

**Purpose**: Represents a decomposed task plan generated by LLM.

**Attributes**:
- `id` (string): Unique plan identifier (UUID)
- `timestamp` (datetime): When plan was generated
- `voice_command_id` (string): Reference to originating VoiceCommand
- `goal` (string): High-level goal from user
- `steps` (list[ExecutionStep]): Ordered steps to execute
- `metadata` (dict): LLM reasoning, alternatives considered, confidence
- `estimated_duration_seconds` (float): Predicted execution time

**Validation Rules**:
- `steps` must contain 1-20 items (guard against infinite loops)
- Each step must have a valid `action_type` (navigate, perceive, manipulate, request_clarification)
- `estimated_duration_seconds` must be > 0

**State Transitions**:
- `GENERATED` → `EXECUTING` → `COMPLETED` or `FAILED`

**Example**:
```json
{
  "id": "plan-001",
  "timestamp": "2026-01-26T10:30:02Z",
  "voice_command_id": "voice-cmd-001",
  "goal": "bring me the red cube from the table",
  "steps": [
    {
      "step_index": 1,
      "action_type": "navigate",
      "parameters": {"target_location": "kitchen_table", "approach_distance": 0.5},
      "expected_outcome": "robot at kitchen_table"
    },
    {
      "step_index": 2,
      "action_type": "perceive",
      "parameters": {"object_class": "cube", "color": "red"},
      "expected_outcome": "object_pose estimated"
    },
    {
      "step_index": 3,
      "action_type": "manipulate",
      "parameters": {"action": "grasp", "object_id": "red_cube"},
      "expected_outcome": "red_cube in gripper"
    },
    {
      "step_index": 4,
      "action_type": "navigate",
      "parameters": {"target_location": "user_location"},
      "expected_outcome": "robot at user"
    }
  ],
  "metadata": {
    "llm_model": "gpt-4",
    "confidence": "high",
    "reasoning": "Decomposed task into sequential navigation and manipulation steps"
  },
  "estimated_duration_seconds": 25
}
```

**ROS 2 Message Equivalent**:
```yaml
# custom_msgs/TaskPlan.msg
string id
builtin_interfaces/Time timestamp
string voice_command_id
string goal
ExecutionStep[] steps
string metadata_json  # JSON string for flexibility
float64 estimated_duration_seconds
```

### 3. ExecutionStep

**Purpose**: Represents a single actionable step within a TaskPlan.

**Attributes**:
- `step_index` (int): Order in execution (1-indexed)
- `action_type` (string): One of ["navigate", "perceive", "manipulate", "request_clarification"]
- `parameters` (dict): Action-specific parameters
  - For `navigate`: target_location, approach_distance, timeout
  - For `perceive`: object_class, color, confidence_threshold
  - For `manipulate`: action (grasp/release), object_id, force
  - For `request_clarification`: question, options
- `expected_outcome` (string): What success looks like
- `timeout_seconds` (int): Max time to wait (default: 30s)

**Validation Rules**:
- `action_type` must be one of the four defined types
- `parameters` must match action_type schema
- `timeout_seconds` must be 1-120

**State Transitions**:
- `PENDING` → `EXECUTING` → `SUCCESS` or `FAILED` or `TIMEOUT`

**Example** (navigate step):
```json
{
  "step_index": 1,
  "action_type": "navigate",
  "parameters": {
    "target_location": "kitchen_table",
    "approach_distance": 0.5,
    "timeout": 30
  },
  "expected_outcome": "robot at kitchen_table within 0.1m",
  "timeout_seconds": 30
}
```

**ROS 2 Message Equivalent**:
```yaml
# custom_msgs/ExecutionStep.msg
int32 step_index
string action_type
string parameters_json
string expected_outcome
int32 timeout_seconds
```

---

### 4. ExecutionTrace

**Purpose**: Records the complete execution of a TaskPlan, including outcomes and latencies.

**Attributes**:
- `id` (string): Unique trace identifier (UUID)
- `plan_id` (string): Reference to executed TaskPlan
- `start_timestamp` (datetime): When execution started
- `end_timestamp` (datetime): When execution completed
- `total_duration_seconds` (float): Actual execution time
- `executed_steps` (list[ExecutedStep]): Results for each step
- `success_rate` (float): Percentage of steps completed successfully (0.0-1.0)
- `failure_reason` (string, optional): If failed, why
- `safety_incidents` (int): Count of collisions or errors
- `logs` (list[string]): Debug/diagnostic logs

**Validation Rules**:
- `success_rate` must be 0.0-1.0
- `total_duration_seconds` must be > 0
- If `success_rate` < 1.0, `failure_reason` should be non-empty

**State Transitions**:
- `IDLE` → `ACTIVE` → `COMPLETED` or `ABORTED`

**Example**:
```json
{
  "id": "trace-001",
  "plan_id": "plan-001",
  "start_timestamp": "2026-01-26T10:30:02Z",
  "end_timestamp": "2026-01-26T10:30:27Z",
  "total_duration_seconds": 25.3,
  "executed_steps": [
    {
      "step_index": 1,
      "action_type": "navigate",
      "status": "SUCCESS",
      "actual_duration_seconds": 8.5,
      "outcome": "reached kitchen_table"
    },
    {
      "step_index": 2,
      "action_type": "perceive",
      "status": "SUCCESS",
      "actual_duration_seconds": 2.1,
      "outcome": "red_cube detected at (1.2, 0.8, 0.9)"
    },
    {
      "step_index": 3,
      "action_type": "manipulate",
      "status": "SUCCESS",
      "actual_duration_seconds": 4.2,
      "outcome": "gripper closed, object secured"
    },
    {
      "step_index": 4,
      "action_type": "navigate",
      "status": "SUCCESS",
      "actual_duration_seconds": 9.8,
      "outcome": "returned to user"
    }
  ],
  "success_rate": 1.0,
  "safety_incidents": 0,
  "logs": [
    "2026-01-26T10:30:02Z: Plan execution started",
    "2026-01-26T10:30:10Z: Navigation step 1 complete",
    "2026-01-26T10:30:27Z: Plan execution finished, all steps succeeded"
  ]
}
```

**ROS 2 Message Equivalent**:
```yaml
# custom_msgs/ExecutionTrace.msg
string id
string plan_id
builtin_interfaces/Time start_timestamp
builtin_interfaces/Time end_timestamp
float64 total_duration_seconds
ExecutedStep[] executed_steps
float64 success_rate
string failure_reason
int32 safety_incidents
string[] logs
```

---

### 5. ExecutedStep

**Purpose**: Result record for a single executed step (nested within ExecutionTrace).

**Attributes**:
- `step_index` (int): Which step in the plan
- `action_type` (string): Type of action
- `status` (string): One of ["SUCCESS", "FAILED", "TIMEOUT", "SKIPPED"]
- `actual_duration_seconds` (float): How long it actually took
- `outcome` (string): What actually happened (vs. expected)
- `error_message` (string, optional): If failed, error details

**Validation Rules**:
- `status` must be one of the four defined values
- `actual_duration_seconds` must be >= 0

**Example**:
```json
{
  "step_index": 2,
  "action_type": "perceive",
  "status": "SUCCESS",
  "actual_duration_seconds": 2.1,
  "outcome": "red_cube detected at (1.2, 0.8, 0.9)",
  "error_message": null
}
```

## Relationships

```
VoiceCommand
    ↓ (originates)
TaskPlan
    ↓ (executed as)
ExecutionTrace
    ├── ExecutedStep[1]
    ├── ExecutedStep[2]
    ├── ExecutedStep[3]
    └── ExecutedStep[N]
```

## ROS 2 Message Files

Place the following `.msg` files in a new ROS 2 package `custom_msgs`:

**Location**: `src/custom_msgs/msg/`

Files to create:
- `VoiceCommand.msg`
- `TaskPlan.msg`
- `ExecutionStep.msg`
- `ExecutedStep.msg`
- `ExecutionTrace.msg`

**CMakeLists.txt** snippet:
```cmake
find_package(builtin_interfaces REQUIRED)
find_package(rosidl_default_generators REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/VoiceCommand.msg"
  "msg/TaskPlan.msg"
  "msg/ExecutionStep.msg"
  "msg/ExecutedStep.msg"
  "msg/ExecutionTrace.msg"
  DEPENDENCIES builtin_interfaces
)
```

## Python Dataclasses (For Examples)

For non-ROS 2 contexts (e.g., local testing), use Python dataclasses:

```python
# module4_common/data_models.py

from dataclasses import dataclass, field
from datetime import datetime
from typing import List, Dict, Optional

@dataclass
class VoiceCommand:
    id: str
    timestamp: datetime
    transcript: str
    confidence_score: float
    duration_seconds: float
    language: str = "en"
    model_version: str = "base"
    raw_audio_path: Optional[str] = None

@dataclass
class ExecutionStep:
    step_index: int
    action_type: str  # "navigate", "perceive", "manipulate", "request_clarification"
    parameters: Dict
    expected_outcome: str
    timeout_seconds: int = 30

@dataclass
class TaskPlan:
    id: str
    timestamp: datetime
    voice_command_id: str
    goal: str
    steps: List[ExecutionStep]
    metadata: Dict = field(default_factory=dict)
    estimated_duration_seconds: float = 30.0

@dataclass
class ExecutedStep:
    step_index: int
    action_type: str
    status: str  # "SUCCESS", "FAILED", "TIMEOUT", "SKIPPED"
    actual_duration_seconds: float
    outcome: str
    error_message: Optional[str] = None

@dataclass
class ExecutionTrace:
    id: str
    plan_id: str
    start_timestamp: datetime
    end_timestamp: datetime
    total_duration_seconds: float
    executed_steps: List[ExecutedStep]
    success_rate: float
    safety_incidents: int = 0
    failure_reason: Optional[str] = None
    logs: List[str] = field(default_factory=list)
```

## Summary

The data model supports:
1. **Capture**: VoiceCommand records what was said and with what confidence
2. **Planning**: TaskPlan breaks down goals into executable steps
3. **Execution**: ExecutionTrace records actual outcomes vs. expected
4. **Learning**: Logs enable debugging and student reflection
5. **Integration**: ROS 2 messages for inter-node communication; Python dataclasses for local testing

This structure aligns with Module 1-3 ROS 2 patterns and supports RAG-optimized documentation (semantic chunking by entity).

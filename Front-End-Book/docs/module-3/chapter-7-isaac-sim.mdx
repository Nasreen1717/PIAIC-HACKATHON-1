## 1. Isaac Sim Architecture and Installation

### 1.1 What is Isaac Sim?

**Isaac Sim** is NVIDIA's photorealistic simulation platform built on the Omniverse engine. It enables:

- **Physics-accurate robotics simulation** using NVIDIA PhysX 5.x engine
- **Photorealistic rendering** with RTX ray tracing for realistic perception training
- **GPU-accelerated synthetic data generation** for computer vision datasets
- **ROS 2 integration** for seamless deployment of perception and control algorithms
- **Domain randomization** to close the sim-to-real gap

Isaac Sim replaces Gazebo for high-fidelity robotics applications requiring photorealistic data or physics accuracy beyond traditional simulators.

## 2. URDF to USD Conversion Pipeline

### 2.1 Understanding USD Format

**USD (Universal Scene Description)** is Pixar's open standard for describing 3D scenes:

```
URDF (Robot Description)
    ↓
    └─→ USD (Universal Scene Description)
            ├─ Mesh references (.obj, .usd)
            ├─ Physics properties (mass, friction)
            ├─ Articulation (joints, links)
            └─ Collision shapes (primitives, meshes)
```

**USD Structure Example**:

```usda
#usda 1.0

def Xform "root_link" {
    rel physics:rigidBody = </root_link/RigidBodySchema>

    def Mesh "collision_mesh" {
        asset inputs:chdir = @./meshes/collision.obj@
        bool collision:approximateConvexDecomp = true
    }

    def RigidBodySchema "RigidBodySchema" {
        float3 physics:angularVelocity = (0, 0, 0)
        float3 physics:velocity = (0, 0, 0)
        float physics:mass = 1.0
    }

    def Joint "root_to_joint1" {
        float[] physics:axis = [0, 0, 1]  # Z-axis rotation
        float physics:lowerLimit = -3.14
        float physics:upperLimit = 3.14
    }
}
```

### 2.2 Conversion Process

#### Step 1: Prepare URDF

Ensure URDF includes:
- Correct mesh file paths (relative or absolute)
- Mass and inertia definitions for all links
- Valid collision geometry

```xml
<robot name="humanoid">
  <link name="base_link">
    <inertial>
      <mass value="20.0"/>
      <inertia ixx="0.5" ixy="0" ixz="0" iyy="0.5" iyz="0" izz="0.5"/>
    </inertial>
    <visual>
      <mesh filename="package://humanoid_description/meshes/base_link.obj"/>
    </visual>
    <collision>
      <geometry>
        <mesh filename="package://humanoid_description/meshes/base_link.obj"/>
      </geometry>
    </collision>
  </link>
</robot>
```

#### Step 2: Convert with Isaac Sim Script

```python
#!/usr/bin/env python3
"""Convert URDF to USD."""

from omni.isaac.core.utils.stage import add_reference_to_stage
from omni.isaac.core.utils.nucleus import find_nucleus_server
import omni

async def convert_urdf_to_usd(urdf_path: str, usd_output_path: str):
    """🔄 Convert URDF to USD format."""

    # Import URDF
    from omni.importer.urdf import USDHelpers

    importer = USDHelpers()
    importer.convert_urdf_to_usd(
        urdf_file_path=urdf_path,
        usd_file_path=usd_output_path,
        # Physics configuration
        fix_base=False,  # Allow base link to move
        invert_y_z=False,
        decompose_meshes=True,  # Split complex meshes
        self_collision=False,
        merge_fixed_joints=True
    )

    print(f"✅ Converted: {urdf_path} → {usd_output_path}")

# Usage
convert_urdf_to_usd(
    "/path/to/robot.urdf",
    "/path/to/robot.usd"
)
```

### 2.3 Physics Configuration in USD

**Joint Articulation** (example):

```python
from omni.isaac.core.prims import RigidPrim, ArticulationPrim

# Load USD with articulation
robot = ArticulationPrim(
    prim_path="/World/robot",
    name="robot"
)

# Access joint
joint = robot.get_joint_position_targets()
robot.set_joint_position_targets(joint_positions=[0.5, 0.3, 0.2])

# Set joint limits
for joint in robot.get_joints():
    joint.set_limits(
        lower_limit=-3.14,
        upper_limit=3.14,
        effort=100,
        velocity=1.0
    )
```

**Collision Mesh Handling**:
- Visual meshes: High-poly for rendering (1M+ triangles OK)
- Collision meshes: Low-poly for physics (~10k triangles per link)
- Decompose complex meshes into convex shapes for stability

## 3. Physics Engine Configuration

### 3.1 PhysX Engine Parameters

Isaac Sim uses **NVIDIA PhysX 5.x**, configurable at multiple levels:

```python
from omni.isaac.core.simulation_context import SimulationContext

# Configure global physics
sim_context = SimulationContext(
    physics_dt=0.01,  # Simulation timestep (10ms)
    rendering_dt=0.016,  # Render timestep (60fps)
    backend="cuda"  # Use GPU acceleration
)

# Access physics scene
physics_scene = sim_context.get_physics_context()

# Set gravity
physics_scene.set_gravity(9.81)

# Configure solver iterations
physics_scene.set_solver_iterations(4)  # Default: 4, Higher = more stable
physics_scene.set_solver_velocity_iterations(1)

# Enable GPU acceleration
physics_scene.enable_gpu_dynamics(True)
```

### 3.2 Material Properties

**Friction and Restitution**:

```python
from omni.isaac.core.materials import PhysicsMaterial

# Define material
material = PhysicsMaterial(
    static_friction=0.7,      # 0.0=slippery, 1.0=sticky
    dynamic_friction=0.5,
    restitution=0.2           # 0.0=no bounce, 1.0=perfect bounce
)

# Apply to rigid body
prim = stage.GetPrimAtPath("/World/robot/base_link")
prim.GetAttribute("physics:material").Set(material)
```

**Recommended Values by Surface**:

| Surface Type | Static Friction | Dynamic Friction | Restitution |
|--------------|-----------------|------------------|-------------|
| **Wood on Wood** | 0.5-0.6 | 0.4-0.5 | 0.3-0.5 |
| **Metal on Metal** | 0.3-0.4 | 0.2-0.3 | 0.1-0.3 |
| **Rubber on Floor** | 0.7-0.9 | 0.6-0.8 | 0.1-0.2 |
| **Humanoid Feet** | 0.8 | 0.7 | 0.1 |

### 3.3 Timestep and Stability

```python
# Timestep selection rules:
# 1. Lower timestep = more accurate but slower
# 2. Rule of thumb: dt < 0.01 for most robots
# 3. For fast motion: dt = 0.001 (1kHz)

physics_scene.set_physics_dt(0.005)  # 5ms = 200 Hz

# Convergence iterations (higher = more stable)
physics_scene.set_solver_iterations(8)

# For soft/compliant bodies:
physics_scene.enable_stabilization_iterations(True)
```

**Physics Divergence Detection**:

```python
def check_physics_stability(sim, max_velocity=100.0):
    """⚠️ Detect unstable physics."""
    for prim in sim.get_all_physics_bodies():
        vel = prim.get_linear_velocity()
        if (vel ** 2).sum() ** 0.5 > max_velocity:
            print(f"⚠️  Physics divergence detected on {prim.name}")
            print(f"   Velocity: {vel} (exceeds {max_velocity})")
            return False
    return True
```

### 4.2 Depth Sensor Configuration

**Depth sensor types**:
- Stereo vision: Matches feature points between two cameras
- Structured light: Projects patterns, measures phase shift
- Time-of-flight: Measures light roundtrip time

```python
from omni.isaac.sensor import DepthCamera

depth_camera = DepthCamera(
    prim_path="/World/DepthCamera",
    position=[0, 0, 1],
    resolution=(640, 480),
    min_range=0.1,      # meters
    max_range=5.0,      # meters
    depth_type="linear_depth"  # or "disparity"
)

# Render and get depth
depth_map = depth_camera.get_depth()  # Shape: (480, 640)

# Convert to point cloud
import numpy as np

def depth_to_pointcloud(depth, intrinsics):
    """💾 Convert depth map to 3D points."""
    h, w = depth.shape
    fx, fy, cx, cy = intrinsics['fx'], intrinsics['fy'], intrinsics['cx'], intrinsics['cy']

    x = np.arange(w)
    y = np.arange(h)
    xx, yy = np.meshgrid(x, y)

    z = depth
    x = (xx - cx) * z / fx
    y = (yy - cy) * z / fy

    points = np.stack([x, y, z], axis=-1).reshape(-1, 3)
    return points[~np.isnan(points).any(axis=1)]
```

```python
print(f"Magnetometer: {imu_reading.mag}")

# Sensor noise model (realistic)
class IMUNoise:
    def __init__(self, accel_noise=0.01, gyro_noise=0.001):
        self.accel_noise = accel_noise
        self.gyro_noise = gyro_noise

    def add_noise(self, accel, gyro):
        accel += np.random.normal(0, self.accel_noise, accel.shape)
        gyro += np.random.normal(0, self.gyro_noise, gyro.shape)
        return accel, gyro
```

## 5. Synthetic Data Generation Pipeline

### 5.1 Dataset Generation Workflow

```
Setup Scene
    ↓
Randomize (domain randomization)
    ├─ Object positions, scales, materials
    ├─ Lighting conditions, camera parameters
    ├─ Physics parameters (friction, mass)
    ↓
Run Simulation
    ├─ Physics step
    ├─ Capture sensor data (RGB, depth, semantic)
    ├─ Record pose ground truth
    ↓
Annotate
    ├─ 2D bounding boxes
    ├─ 3D object poses
    ├─ Semantic segmentation
    ↓
Export
    └─ COCO format, Pascal VOC, or custom
```

## 7. Troubleshooting Common Issues

### 7.1 GPU Memory Errors

```
Error: CUDA out of memory
```

**Solution**:
1. Reduce camera resolution (1920×1080 → 1280×720)
2. Decrease scene complexity (fewer objects, simpler meshes)
3. Enable GPU memory garbage collection: `physics_scene.enable_memory_optimization(True)`

### 7.2 Physics Divergence

```
Bodies flying apart, extreme velocities
```

**Solution**:
1. Reduce timestep: 0.01 → 0.005
2. Increase solver iterations: 4 → 8
3. Check joint limits and friction values
4. Disable self-collision for low-mass links

### 7.3 Mesh Import Failures

```
Could not load mesh file
```

**Solution**:
1. Verify mesh file path is absolute or relative to URDF package
2. Use supported formats: .obj, .glb, .fbx
3. Ensure mesh is non-degenerate (check with Meshlab)

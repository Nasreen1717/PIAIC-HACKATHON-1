## 1. Natural Language to Executable Plans

### 1.1 The Task Decomposition Problem

**Task Decomposition** is the cognitive process of breaking down a high-level goal into executable steps. For example:

**Goal**: "Bring me the red cube from the kitchen table"

**Decomposed Plan**:
1. Navigate to kitchen (parameter: location="kitchen")
2. Perceive objects on table (parameter: location="table", target="red cube")
3. Grasp the red cube (parameter: object_id=&lt;from step 2&gt;)
4. Navigate back to user (parameter: location="user_location")
5. Release object to user (parameter: object_id=&lt;from step 3&gt;)

**Cognitive Pipeline**:

```
┌──────────────────────────┐
│  Natural Language Goal   │
│  "bring me red cube..."  │
└────────┬─────────────────┘
         │
         ↓
┌──────────────────────────────────┐
│  LLM (GPT-4) Analysis            │
│  • Understand user intent        │
│  • Extract constraints & context │
│  • Decompose into steps          │
└────────┬─────────────────────────┘
         │
         ↓
┌──────────────────────────────────┐
│  Function Calling                │
│  • Map steps to robot actions    │
│  • Generate JSON parameters      │
│  • Validate against schema       │
└────────┬─────────────────────────┘
         │
         ↓
┌──────────────────────────────────┐
│  Plan Validation                 │
│  • Check step ordering           │
│  • Verify action types           │
│  • Test parameter constraints    │
└────────┬─────────────────────────┘
         │
         ↓
┌──────────────────────────────────┐
│  Executable Task Plan            │
│  [Navigation, Perception, ...]   │
└──────────────────────────────────┘
```

### 1.2 LLM Confidence & Limitations

**When LLMs Succeed**:
- Clear, well-specified goals ("pick up the red object on the table")
- Tasks within robot's known capabilities (navigation, grasping, perception)
- Sufficient context provided (room layout, object descriptions, success criteria)

**When LLMs Struggle**:
- Ambiguous goals ("help me with the task" - what task?)
- Impossible requests ("fly to the moon" - humanoid can't fly)
- Underspecified parameters ("move the small thing" - which thing? where to?)
- Contradictory constraints ("move fast and carefully" - conflicting)

**Confidence Rating**:

| Rating | Indicator | Action |
|--------|-----------|--------|
| **High** | Single clear decomposition, all steps executable | Execute plan immediately |
| **Medium** | Multiple possible interpretations, some steps unclear | Request clarification on key parameter |
| **Low** | Goal ambiguous, missing critical information | Ask user to rephrase with more detail |

## 2. LLM Function Calling Architecture

### 2.1 Function Calling Basics

**Function Calling** allows an LLM to invoke external tools by returning structured JSON. Instead of just text, the LLM returns:

```json
{
  "function": "navigate",
  "parameters": {
    "destination": "kitchen",
    "speed": "normal",
    "avoid_obstacles": true
  }
}
```

**GPT-4 Function Calling Flow**:

```
1. Define robot capabilities as JSON schema
2. Submit goal + schema to GPT-4 API
3. GPT-4 returns function calls (structured JSON)
4. Parse and validate function calls
5. Execute functions in order
6. Return results to GPT-4 (for next step)
```

### 2.2 Defining Robot Capabilities

**Robot Capability Schema** describes what a robot can do:

```yaml

      - name: speed

        default: "normal"
        enum: ["slow", "normal", "fast"]

        type: boolean
        default: true

  - name: perceive
    description: "Detect and identify objects in view"

      - name: location

      - field: object_id
        type: string
        description: "Unique identifier for found object"
```

## 3. Robot Capability Modeling

### 3.1 Action Schema Design

Each robot action has a **schema** that defines:
- Input parameters (with types, ranges, constraints)
- Output/return values
- Success criteria
- Timeout/duration expectations

**Example: Navigate Action Schema**:

```json
{
  "name": "navigate",
  "schema": {
    "type": "object",
    "properties": {
      "destination": {
        "type": "string",
        "enum": ["kitchen", "bedroom", "living_room", "hallway"],
        "description": "Target location"
      },
      "speed": {
        "type": "string",
        "enum": ["slow", "normal", "fast"],
        "default": "normal"
      },
      "avoid_obstacles": {
        "type": "boolean",
        "default": true
      }
    },
    "required": ["destination"]
  },
  "expected_duration": 5.0,
  "success_criteria": "Robot reaches destination",
  "failure_modes": ["path_blocked", "collision_detected", "timeout"]
}
```

### 3.2 Parameter Validation

**Validation Rules** ensure parameters are valid before execution:

```python
class ParameterValidator:
    """Validate action parameters against schema."""

    def validate(self, action_name: str, parameters: dict) -> tuple[bool, str]:
        """
        Validate parameters.

        Returns:
            (is_valid, error_message)
        """
        schema = self.get_schema(action_name)

        # ✅ Check required fields
        for required_field in schema['required']:
            if required_field not in parameters:
                return False, f"Missing required parameter: {required_field}"

        # ✅ Check field types and constraints
        for param_name, param_value in parameters.items():
            if param_name not in schema['properties']:
                return False, f"Unknown parameter: {param_name}"

            prop_schema = schema['properties'][param_name]

            # Check type
            if prop_schema['type'] == 'string' and not isinstance(param_value, str):
                return False, f"Parameter {param_name} must be string"

            # Check enum constraints
            if 'enum' in prop_schema and param_value not in prop_schema['enum']:
                return False, f"Parameter {param_name}={param_value} not in {prop_schema['enum']}"

        return True, ""
```

## 4. Plan Validation & Safety Checks

### 4.1 Plan Structure Validation

**Valid plans** have specific properties:

| Property | Constraint | Example |
|----------|-----------|---------|
| **Number of steps** | 1-20 | [navigate, perceive, manipulate] = 3 ✅ |
| **Step ordering** | Logical sequence | perceive BEFORE manipulate ✅ |
| **Action types** | Valid robot actions | [navigate, perceive, manipulate, request_clarification] |
| **Parameters** | Match schema | destination in ["kitchen", "bedroom", ...] |
| **Duration** | Sum ≤ 30 seconds | 5s + 2s + 3s = 10s < 30s ✅ |

### 4.2 Validation Logic

```python
class PlanValidator:
    """Validate generated task plans."""

    def validate_plan(self, plan: dict) -> tuple[bool, list]:
        """
        Validate complete plan.

        Returns:
            (is_valid, error_list)
        """
        errors = []

        # ✅ Check step count
        steps = plan.get('steps', [])
        if len(steps) == 0:
            errors.append("Plan has no steps")
        elif len(steps) > 20:
            errors.append(f"Plan has too many steps: {len(steps)} > 20")

        # ✅ Validate each step
        total_duration = 0
        for i, step in enumerate(steps):
            action_type = step.get('action_type')

            # Validate action type
            if action_type not in ['navigate', 'perceive', 'manipulate', 'request_clarification']:
                errors.append(f"Step {i}: Invalid action_type: {action_type}")

            # Validate parameters
            is_valid, error = self.validate_parameters(action_type, step.get('parameters', {}))
            if not is_valid:
                errors.append(f"Step {i}: {error}")

            # Accumulate duration
            duration = step.get('timeout_seconds', 5.0)
            total_duration += duration

        # ✅ Check total duration
        if total_duration > 30.0:
            errors.append(f"Total plan duration {total_duration}s exceeds 30s limit")

        # ✅ Check step dependencies
        for i, step in enumerate(steps):
            if step['action_type'] == 'manipulate':
                # Manipulate should follow perceive (to get object_id)
                if i == 0 or all(s['action_type'] != 'perceive' for s in steps[:i]):
                    errors.append(f"Step {i}: manipulate should follow perceive")

        return len(errors) == 0, errors
```

### 4.3 Safety Constraints

**Safety Rules** prevent dangerous plans:

```python
class SafetyChecker:
    """Check plan for safety violations."""

    SAFE_SPEED_THRESHOLDS = {
        "slow": 0.1,      # m/s
        "normal": 0.5,    # m/s
        "fast": 1.0       # m/s
    }

    def check_safety(self, plan: dict) -> tuple[bool, list]:
        """
        Check plan for safety violations.

        Returns:
            (is_safe, warning_list)
        """
        warnings = []

        for i, step in enumerate(plan['steps']):
            action = step['action_type']
            params = step.get('parameters', {})

            # ⚠️ Check navigation speed near obstacles
            if action == 'navigate':
                speed = params.get('speed', 'normal')
                if not params.get('avoid_obstacles', True) and speed == 'fast':
                    warnings.append(
                        f"Step {i}: Fast navigation with obstacle avoidance disabled"
                    )

            # ⚠️ Check gripper force for delicate objects
            if action == 'manipulate':
                obj_type = params.get('object_type', 'unknown')
                if obj_type in ['fragile', 'electronics'] and params.get('grip_force', 5) > 3:
                    warnings.append(
                        f"Step {i}: High grip force ({params['grip_force']}) on fragile object"
                    )

        return len(warnings) == 0, warnings
```

## 5. Error Handling & Ambiguity Resolution

### 5.1 Ambiguity Detection

**Ambiguous Goals** lack critical information:

| Goal | Issues | Fix |
|------|--------|-----|
| "Pick up the small thing" | Which object? Where? | "Pick up the small red cube on the kitchen table" |
| "Move it somewhere" | Move what? Where? | "Move the cube to the bedroom" |
| "Help with the task" | What task? | "Move the red cube from table to shelf" |

**Detection Strategy**:
```python
def detect_ambiguity(goal: str) -> tuple[bool, list]:
    """
    Detect ambiguity in goal statement.

    Returns:
        (is_ambiguous, ambiguity_list)
    """
    ambiguities = []

    # ✅ Check for pronouns without antecedents
    if re.search(r'\b(it|them|that)\b', goal, re.IGNORECASE):
        ambiguities.append("Pronoun reference unclear (it, them, that)")

    # ✅ Check for vague object references
    if re.search(r'\b(thing|stuff|object|item)\b', goal, re.IGNORECASE):
        ambiguities.append("Object reference is vague")

    # ✅ Check for incomplete location info
    if re.search(r'\b(somewhere|somewhere|anywhere)\b', goal, re.IGNORECASE):
        ambiguities.append("Destination is unspecified")

    return len(ambiguities) > 0, ambiguities
```

### 5.2 Request Clarification

When ambiguity detected, use a special action to ask user:

```python
clarification_response = {
    "function": "request_clarification",
    "parameters": {
        "question": "Which object should I pick up?",
        "options": [
            "red cube on the table",
            "blue bottle on the shelf",
            "green mug in the kitchen"
        ]
    }
}
```

### 5.3 Error Recovery Patterns

| Error Type | Recovery Strategy |
|-----------|-------------------|
| **Path Blocked** | Retry navigation with alternate route |
| **Object Not Found** | Expand search area; ask user for location hint |
| **Gripper Slip** | Retry grasp with increased force or different angle |
| **Timeout** | Skip step; ask user to intervene |
| **LLM Rate Limit** | Queue request; retry after delay |

## 6. Plan Quality Metrics

### 6.1 Defining Plan Quality

**High-Quality Plans**:
- ✅ All steps are executable (valid action types, valid parameters)
- ✅ Steps are in logical order (perceive before manipulate)
- ✅ Parameters are realistic (destination in known locations, grip force in valid range)
- ✅ Expected duration is reasonable (&lt;30 seconds for typical task)
- ✅ No redundant steps (navigating to same location twice)

### 6.2 Quality Scoring

```python
class PlanQualityScorer:
    """Score plan quality on 1-10 scale."""

    def score_plan(self, plan: dict) -> int:
        """
        Score plan from 1 (terrible) to 10 (excellent).

        Scoring rubric:
        - Executability: 0-4 points
        - Logical ordering: 0-3 points
        - Realism: 0-3 points
        """
        score = 0

        # ✅ Executability (0-4 points)
        is_valid, errors = self.validate_plan(plan)
        if is_valid:
            score += 4
        else:
            score += max(0, 2 - len(errors))  # Penalize errors

        # ✅ Logical ordering (0-3 points)
        is_logical = self.check_step_ordering(plan)
        if is_logical:
            score += 3

        # ✅ Realism (0-3 points)
        is_realistic = self.check_parameter_realism(plan)
        if is_realistic:
            score += 3

        return max(1, min(10, score))  # Clamp to [1, 10]

    def check_step_ordering(self, plan: dict) -> bool:
        """Check if steps are in logical sequence."""
        steps = plan['steps']

        for i, step in enumerate(steps):
            # Manipulate should follow perceive
            if step['action_type'] == 'manipulate':
                if not any(s['action_type'] == 'perceive' for s in steps[:i]):
                    return False

        return True
```

### 6.3 Measuring Plan Validity Rate

Track how many plans are valid across diverse goals:

```python
def measure_validity_rate(goals: list[str]) -> float:
    """
    Generate plans for diverse goals and measure validity rate.

    Goal: 95%+ valid plans
    """
    planner = LLMPlanner()
    valid_count = 0

    for goal in goals:
        plan = planner.generate_plan(goal)
        is_valid, _ = PlanValidator().validate_plan(plan)
        if is_valid:
            valid_count += 1

    validity_rate = valid_count / len(goals)
    return validity_rate
```

```python
    plan = {
        "goal": goal,
        "steps": plan_steps,
        "estimated_duration_seconds": len(plan_steps) * 3.0  # Rough estimate
    }

    return plan
```

### Example 5: Complex Multi-Step Planning

```python
"""
🚀 Module 4 Example 5: Complex Multi-Step Task Decomposition
Demonstrates decomposition of complex, multi-step goals.
Includes validation and quality scoring.
"""

import json
import time
from llm_client import LLMPlannerClient
from plan_validator import PlanValidator
from data_models import TaskPlan

def complex_task_planning(goal: str) -> dict:
    """
    Generate and validate complex task plan.

    Returns:
        {
            'goal': str,
            'plan': TaskPlan,
            'is_valid': bool,
            'quality_score': int,
            'generation_latency': float,
            'issues': list
        }
    """
    planner = LLMPlannerClient()
    validator = PlanValidator()

    # ✅ Generate plan
    start_time = time.time()
    plan = planner.generate_plan(goal)
    generation_latency = time.time() - start_time

    # ✅ Validate plan
    is_valid, errors = validator.validate_plan(plan)

    # ✅ Score quality
    quality_score = validator.score_plan(plan) if is_valid else 1

    # ✅ Compile results
    result = {
        'goal': goal,
        'plan': plan,
        'is_valid': is_valid,
        'quality_score': quality_score,
        'generation_latency': generation_latency,
        'issues': errors
    }

    return result


def evaluate_goal_diversity(goals: list[str]) -> dict:
    """
    Evaluate planner across diverse goals.
    Tests simple, complex, ambiguous, and impossible goals.
    """
    results = {
        'simple': [],
        'complex': [],
        'ambiguous': [],
        'impossible': [],
        'summary': {}
    }

    # ✅ Simple goals
    simple_goals = [
        "Move forward",
        "Stop",
        "Turn around"
    ]
    for goal in simple_goals:
        result = complex_task_planning(goal)
        results['simple'].append(result)

    # ✅ Complex goals
    complex_goals = [
        "Bring me the red cube from the kitchen table to the living room",
        "Find the blue bottle on the shelf, avoid the cat, and place it on the counter"
    ]
    for goal in complex_goals:
        result = complex_task_planning(goal)
        results['complex'].append(result)

    # ✅ Ambiguous goals
    ambiguous_goals = [
        "Pick up the thing",
        "Move it somewhere",
        "Help with the task"
    ]
    for goal in ambiguous_goals:
        result = complex_task_planning(goal)
        results['ambiguous'].append(result)

    # ✅ Impossible goals
    impossible_goals = [
        "Fly to the moon",
        "Phase through walls"
    ]
    for goal in impossible_goals:
        result = complex_task_planning(goal)
        results['impossible'].append(result)

    # ✅ Compute summary statistics
    all_results = (
        results['simple'] + results['complex'] +
        results['ambiguous'] + results['impossible']
    )
    valid_count = sum(1 for r in all_results if r['is_valid'])

    results['summary'] = {
        'total_goals': len(all_results),
        'valid_plans': valid_count,
        'validity_rate': valid_count / len(all_results),
        'avg_quality_score': sum(r['quality_score'] for r in all_results if r['is_valid']) / max(1, valid_count),
        'avg_latency': sum(r['generation_latency'] for r in all_results) / len(all_results)
    }

    return results


if __name__ == "__main__":
```

```python
🚀 Module 4 Example 6: Ambiguity Detection and Clarification
Demonstrates handling of ambiguous goals and error recovery.

        self.llm_client = LLMPlannerClient()
        self.openai = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

    def detect_ambiguity(self, goal: str) -> tuple[bool, list]:
        """
        ✅ Detect ambiguity patterns in goal.

        Returns:
            (is_ambiguous, issues)
        """
        issues = []

        # Check for vague object references
        if re.search(r'\b(thing|stuff|object|item|it|them)\b', goal, re.IGNORECASE):
            issues.append("Vague object reference")

        # Check for unspecified locations
        if re.search(r'\b(somewhere|anywhere|here|there)\b', goal, re.IGNORECASE):
            issues.append("Location unspecified")

        # Check for unclear actions
        if re.search(r'\b(help|task|do something|assist)\b', goal, re.IGNORECASE):
            issues.append("Action unclear")

        return len(issues) > 0, issues

    def request_clarification(self, goal: str, issues: list) -> str:
        """
        🤔 Ask user for clarification on ambiguous goal.

        Returns:
            Clarified goal statement
        """
        print(f"❓ Your goal '{goal}' is ambiguous.")
        print(f"   Issues: {', '.join(issues)}")

        # Generate clarification questions via LLM
        prompt = f"""
The user said: '{goal}'
The issues with this are: {issues}

Generate 3 clarifying questions to help the user be more specific.
Return as a numbered list.
"""

        response = self.openai.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.3
        )

        print(f"\n   Please clarify:")
        print(response.choices[0].message.content)

        # Simulate user input (in real system, prompt user)
        clarified_goal = input("\n   Clarified goal: ").strip()
        return clarified_goal

    def handle_impossible_goal(self, goal: str) -> dict:
        """
        ❌ Detect and handle impossible goals.

        Returns:
            {'status': 'impossible', 'reason': str, 'alternatives': list}
        """
        impossible_patterns = [
            (r'\b(fly|float|levitate)\b', "Robot cannot fly"),
            (r'\b(phase through|teleport|disappear)\b', "Robot cannot phase through objects"),
            (r'\b(destroy|break|smash)\b', "Robot should not destroy objects"),
            (r'\b(moon|space|universe)\b', "Destination is out of reach")
        ]

        for pattern, reason in impossible_patterns:
            if re.search(pattern, goal, re.IGNORECASE):
                return {
                    'status': 'impossible',
                    'reason': reason,
                    'goal': goal,
                    'suggestion': "Please rephrase using achievable robot capabilities: navigate, perceive, manipulate, request_clarification"
                }

        return {'status': 'possible'}

    def process_goal_with_error_handling(self, goal: str) -> dict:
        """
        ✅ Full goal processing with error handling.

        Returns:
            {
                'status': 'success' | 'clarification_needed' | 'impossible',
                'goal': str,
                'plan': TaskPlan or None,
                'action': str
            }
        """
        # ✅ Check for impossible goals
        impossible_check = self.handle_impossible_goal(goal)
        if impossible_check['status'] == 'impossible':
            return {
                'status': 'impossible',
                'goal': goal,
                'reason': impossible_check['reason'],
                'suggestion': impossible_check.get('suggestion')
            }

        # ✅ Check for ambiguous goals
        is_ambiguous, issues = self.detect_ambiguity(goal)
        if is_ambiguous:
            clarified_goal = self.request_clarification(goal, issues)
            return {
                'status': 'clarification_needed',
                'original_goal': goal,
                'clarified_goal': clarified_goal,
                'action': 'resubmit with clarified goal'
            }

        # ✅ Generate plan
        plan = self.llm_client.generate_plan(goal)

        return {
            'status': 'success',
            'goal': goal,
            'plan': plan
        }


if __name__ == "__main__":
```
